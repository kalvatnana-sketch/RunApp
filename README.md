# NoTraceRun

Cyberpunk / spy-themed Flutter app with **Firebase Authentication** and a **location-based map mode**.

## What it does

- **Secure login + registration**
  - Email/password auth
  - Google Sign-In
  - Password reset
- **Mission Hub**
  - Entry point after login with quick access to modes + settings
- **Story Mode**
  - Mission ladder UI (demo missions / unlock progression)
- **Open Mode (Map)**
  - Uses your device location
  - Shows a dark map with nearby POIs (fuel, restaurants, cafés, hospitals, police, banks, supermarkets, pharmacies, parking)
  - POIs are fetched from **OpenStreetMap via Overpass API**

## Tech stack

- **Flutter** (Dart, Material, dark theme UI)
- **Firebase**: `firebase_core`, `firebase_auth`
- **Google Sign-In**: `google_sign_in`
- **Maps + location**: `flutter_map`, `geolocator`, `latlong2`
- **HTTP**: `http`

## Prerequisites

- Flutter SDK (Dart \(SDK constraint in `pubspec.yaml`: `^3.9.2`\))
- A configured Firebase project (Auth providers enabled)

## Setup

### 1) Install dependencies

```bash
flutter pub get
```

### 2) Firebase configuration

This repo includes `lib/firebase_options.dart` (generated by FlutterFire CLI) and initializes Firebase in `lib/main.dart`.

If you want to connect the app to **your own** Firebase project, regenerate options:

```bash
dart pub global activate flutterfire_cli
flutterfire configure
```

Then ensure the following auth providers are enabled in Firebase Console:

- **Email/Password**
- **Google**

### 3) Platform permissions

- **iOS**: `NSLocationWhenInUseUsageDescription` is set in `ios/Runner/Info.plist`.
- **Android**: location + internet permissions are present in `android/app/src/main/AndroidManifest.xml`.

## Run

```bash
flutter run
```

Common variants:

```bash
flutter run -d ios
flutter run -d android
flutter run -d chrome
```

## App navigation (routes)

Defined in `lib/main.dart`:

- `/` → Welcome
- `/login` → Login
- `/register` → Register
- `/home` → Mission Hub
- `/storyMode` → Story Mode
- `/openMode` → Open Mode (map)
- `/settings` → Settings

## Notes / troubleshooting

- **Google Sign-In on Android** often requires adding your debug/release **SHA-1** to the Firebase project and downloading updated config (the FlutterFire flow handles this, but the SHA-1 still must be registered).
- **Map / POIs require internet** (Overpass API). If POIs don’t appear, confirm network access and that location services are enabled.
